# secure-api-lab

Цей проєкт є реалізацією захищеного REST API, розробленого на базі Node.js та Express. 
Він демонструє принципи побудови RESTful архітектури, включаючи роботу з ресурсами, 
використання HTTP-методів та правильну обробку статусів відповідей.

Основна увага в проєкті приділена безпеці та моніторингу. Реалізовано власні проміжні 
обробники (middleware) для аутентифікації користувачів, авторизації на основі ролей 
(Role-Based Access Control) та логування всіх вхідних запитів.

## Встановлення та запуск

Перед початком роботи потрібно переконатися, що у Node.js встановлено.

1. **Клонуйте репозиторій та перейдіть у папку проєкту:**
   ```bash
   git clone [https://github.com/alisha2112/secure-api-lab.git](https://github.com/YourUsername/secure-api-lab.git)

2. **Встановіть залежності:**
    ```bash
    npm install

3. **Запустіть сервер:**
    ```bash
    npm start

Сервер запуститься за адресою http://localhost:3000.

## Тестування

Для перевірки роботи API передбачено автоматизований скрипт, який емулює запити від 
різних користувачів (успішні та помилкові сценарії).
**Запуск тестового клієнта:**
```bash
npm test
```
Ця команда виконає скрипт test-client.js і виведе результати тестування в консоль.

Для тестування вручну (через Postman) потрібно використовувати наступні дані у 
заголовках X-Login та X-Password:

| Роль  | Логін  | Пароль | Опис                                   |
|-------|--------|-------|----------------------------------------|
| User  | user1  | password123 | Доступ тільки до документів            |
| Admin | admin1 | password123 | Доступ до документів та співробітників |

## Опис API Ендпоінтів

Усі запити до API вимагають наявності заголовків аутентифікації.

| Метод  | URL            | Опис        | Необхідні заголовки  | Приклад тіла запиту  | Можливі коди відповіді   |
|--------|----------------|-------------|----------------------|----------------------|----------|
| GET    | /documents     | Отримання списку всіх документів. Доступно всім авторизованим користувачам. | X-Login, X-Password  | | 200 OK, 401 Unauthorized |
| POST   | /documents     | Створення нового документа. | X-Login, X-Password  | json { "title": "Doc Title", "content": "Text..." } |201 Created, 400 Bad Request, 401 Unauthorized|
| DELETE | /documents/:id | Видалення документа за ID. | X-Login, X-Password  | |204 No Content |
| GET    | /employees     | Отримання списку співробітників. Тільки для адміністраторів. | X-Login, X-Password  ||200 OK, 401 Unauthorized, 403 Forbidden |
